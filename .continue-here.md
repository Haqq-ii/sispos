### Key decisions made in this phase
- `catatanKonsultasi` encrypted with AES-256-GCM BEFORE Prisma write (encrypt.ts)
- AuditLog MUST be in same `prisma.$transaction` as Pemeriksaan INSERT/UPDATE
- `broadcastQueueUpdate` ALWAYS called OUTSIDE `prisma.$transaction`
- Lock-screen: Redis key `kader:{kaderId}:activeMeja` (TTL 24h), NOT localStorage
- Zustand `useKaderMejaStore` has NO persist() — Redis is the truth for lock-screen
- Kader routes: `requireRole('kader', 'ketua_kader')` on all endpoints
- Backend Zod: v3 syntax (`required_error`, not `error`)
- Frontend Zod: v4 syntax (`error:`, not `required_error:`)
- `GET /api/kader/today-slots` added (kader had no endpoint for today's slot listing)
- `GET /api/kader/antrian/:id` added with IDOR guard (wargaId not checked — kader can access any antrian for their posyandu)

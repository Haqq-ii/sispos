## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| JWT → service | puskesmasId enters service as a parameter extracted from req.user!.userId in the route layer — never from client query params |
| User data → Excel cells | namaBalita, namaPosyandu, namaOrangTua, kelurahan pass through safeCell() before writing to worksheet cells |
| Prisma select → export | catatanKonsultasi and rekomendasiAi must never appear in the select object |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Elevation of Privilege | generateLaporanBulananXlsx/Pdf — puskesmasId param | mitigate | puskesmasId is a function parameter received from route handler which extracts it from req.user!.userId (JWT); service never reads from req.query or req.body |
| T-05-02 | Tampering | ExcelJS addRow — namaBalita, namaOrangTua, namaPosyandu, kelurahan string cells | mitigate | safeCell() applied to ALL string columns before addRow; pattern /^[=+\-@]/ prefixes with apostrophe |
| T-05-03 | Tampering | format query param validation | mitigate | Validated in route handler (Plan 02) as enum ['xlsx','pdf']; service receives format as typed enum |
| T-05-04 | Information Disclosure | Prisma select — catatanKonsultasi, rekomendasiAi | mitigate | Explicit select object in findMany; these two fields are NOT listed; verified by grep acceptance criterion |
| T-05-SC | Tampering | npm/pip/cargo installs | accept | No new packages in Phase 5; ExcelJS and pdfkit already installed and audited in Phase 3 |
</threat_model>

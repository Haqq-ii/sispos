Apply these four targeted edits to prisma/schema.prisma (edit in place, preserve all other content):

FIX 1 — Add missing back-relation to Warga model.
In the Warga model, after the `antrian Antrian[]` line, add:
  riwayatChat   RiwayatChat[]

FIX 2 — Remove polymorphic relations from AuditLog model.
Delete the two lines at the bottom of the AuditLog model that read:
  puskesmas     Puskesmas?   @relation(fields: [userId], references: [id])
  kader         Kader?       @relation(fields: [userId], references: [id])
Also delete the comment line above them: `// Relasi opsional untuk traceability`
Reason: Prisma cannot use a single scalar field (userId) to reference two different models. AuditLog is an append-only log; userId + userRole is sufficient for lookup.

FIX 3 — Remove broken back-relations from Puskesmas and Kader models.
In the Puskesmas model, delete the line: `auditLog       AuditLog[]`
In the Kader model, delete the line: `auditLog      AuditLog[]`
These were the inverse side of the now-removed relations in AuditLog.

FIX 4 — Add missing Pemeriksaan columns (they are in the schema comments at the bottom but not in the model).
In the Pemeriksaan model, after the `rekomendasiAi` field and before `tanggalPemeriksaan`, add:
  catatanKlinis       String?     // catatan klinis oleh kader (plaintext, bukan PDP-sensitif)
  tandaKlinis         Json?       // { rambutKemerahan: bool, perutBuncit: bool, edema: bool, pucat: bool, lainnya: string | null }
  statusGiziOverride  StatusGizi? // override manual kader jika tidak setuju hasil otomatis

After applying all four fixes, also delete the comment block at the very bottom of the file (the block starting with `// UPDATE: tambah kolom ke model Pemeriksaan yang sudah ada`) — those ALTER TABLE comments are now superseded by the actual model changes above.

Do NOT change any other model, enum, @@map, @unique, @index, or @default — the schema is declared final v2.0.

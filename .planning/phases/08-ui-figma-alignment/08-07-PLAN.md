New backend endpoint POST /api/kader/verify-ketua-pin:
- Authenticated kader submits the ketua's PIN
- Backend finds the ketua_kader (isKetua=true, isAktif=true) of the kader's posyandu
- bcrypt.compare(pin, ketua.pinHash)
- 200 OK on match, 403 KETUA_PIN_SALAH on mismatch

- Does NOT log the submitted PIN value in Pino

- 6-digit PIN input
- Calls POST /api/kader/verify-ketua-pin

- On 403: shows inline error message

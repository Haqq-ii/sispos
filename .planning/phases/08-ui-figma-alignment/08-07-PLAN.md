## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Kader browser → POST /api/kader/verify-ketua-pin | Kader submits another kader's PIN over network |
| Frontend state → 3-attempt lockout | Attempt counter in React state; can be reset by page reload |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-07-01 | Information Disclosure | PIN in POST body | mitigate | Nginx TLS 1.3 in transit; PIN never logged (explicit constraint in action); only bcrypt hash compared server-side |
| T-08-07-02 | Spoofing | verify-ketua-pin without auth | mitigate | authMiddleware + requireRole(['kader','ketua_kader']); citizen and puskesmas JWT rejected |
| T-08-07-03 | Tampering | Frontend-only 3-attempt lockout bypassed by reload | accept | Master Overrule via Puskesmas is the escalation path per CLAUDE.md; bcrypt makes brute force impractical; Posyandu context accepts this risk level |
| T-08-07-04 | Information Disclosure | slotId in navigate state on success | accept | slotId is not a secret; kader already has access to this session |
| T-08-07-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed in this plan |
</threat_model>

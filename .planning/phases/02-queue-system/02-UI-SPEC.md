## Typography

Extends Phase 1. Exactly 4 sizes, 2 weights.

| Role | Size | Weight | Line Height | Tailwind Class | Usage |
|------|------|--------|-------------|----------------|-------|
| Queue Number | 48px (text-5xl) | 700 (font-bold) | 1.0 | `text-5xl font-bold leading-none` | SINGLE-USE exception — TiketAntrianPage.tsx only; not a reusable scale entry |
| Heading | 20px (text-xl) | 700 (font-bold) | 1.2 | `text-xl font-bold leading-tight` | Screen titles, section headings, card titles, time slot labels |
| Body | 14px (text-sm) | 400 (font-normal) | 1.5 | `text-sm font-normal leading-relaxed` | Descriptions, kuota info, session details, posyandu name |
| Label | 14px (text-sm) | 700 (font-bold) | 1.4 | `text-sm font-bold leading-snug` | Form labels, field captions, badge text |
| Caption | 12px (text-xs) | 400 (font-normal) | 1.4 | `text-xs font-normal leading-snug` | "estimasi dapat berubah" disclaimer, secondary info, timestamps |

**Rules (carried from Phase 1, plus Phase 2 additions):**
- Exactly 4 sizes: 48px (Queue Number exception — TiketAntrianPage.tsx only), 20px (Heading), 14px (Body/Label), 12px (Caption)
- Exactly 2 weights: 400 (font-normal) and 700 (font-bold)
- All UI text in Bahasa Indonesia
- Queue number size (text-5xl) applies to nomor urut ONLY — not to any countdown figure
- Countdown figure uses heading size (text-xl) with font-bold — never display size
- Dates: DD/MM/YYYY; Times: HH:MM WIB; Numbers: commas as decimal (8,5 menit)
- "estimasi" label on countdown: always caption size (text-xs), never omitted

### Screen 6 — Buat Jadwal Dialog (Puskesmas)

**Trigger:** "Buat Jadwal Baru" button on Screen 5
**Component:** `<Dialog>` from shadcn (modal)
**Max width:** 480px

**Form fields (in order):**

1. **Posyandu** — `<Select>`, label "Posyandu", placeholder "Pilih Posyandu"
   - Options: fetched from `GET /api/posyandu`
   - Zod: required string

2. **Tanggal Pelaksanaan** — `<Calendar>` inline date picker (shadcn), label "Tanggal Pelaksanaan"
   - Min date: tomorrow (cannot create jadwal in the past)
   - Disabled: dates that already have a jadwal for the same posyandu
   - Format display: "DD/MM/YYYY"

3. **Estimasi Durasi per Orang** — `<Input type="number">`, label "Estimasi Durasi per Orang (menit)", default value 7
   - `min=5`, `max=30`, `step=1`
   - Helper text (caption): "Sistem akan membuat 3 sesi otomatis dengan kuota floor(60 ÷ estimasi) per sesi"
   - Zod: `z.number().int().min(5).max(30)`

4. **Preview Slot Auto-Generate** — rendered after user fills estimasi field, real-time:
   ```
   Preview sesi (estimasi 7 menit → kuota 8/sesi):
   Sesi 1 · 08:00 – 09:00 · 8 kuota
   Sesi 2 · 09:00 – 10:00 · 8 kuota
   Sesi 3 · 10:00 – 11:00 · 8 kuota
   ```
   - Caption size, bg-green-50, p-3, rounded-md
   - Computed client-side: `kuota = Math.floor(60 / estimasi)`, 3 sesi each 60 menit long
   - Start times: 08:00, 09:00, 10:00 (configurable future phase)

**Dialog footer:**
- "Tutup" — ghost button, closes dialog without action
- "Buat Jadwal" — primary button, submits form, `min-h-[44px]`

**Loading state (submit):** Button disabled + `<Loader2 animate-spin />` + "Membuat jadwal..."

**Success:** Dialog closes, toast notification "Jadwal berhasil dibuat. 3 sesi ter-generate otomatis." (green toast, 4 seconds), table/list refreshes via TanStack Query invalidation.

**Error states:**
- 409 JADWAL_SUDAH_ADA: field error on tanggal "Jadwal untuk posyandu ini pada tanggal tersebut sudah ada."
- Generic server error: `<Alert variant="destructive">` inside dialog

```
  → /puskesmas/jadwal               ← navigation TBD in Phase 4 dashboard
       → Dialog: Buat Jadwal
       → /puskesmas/jadwal/:jadwalId (lihat detail — Phase 3 triggers)
```

### Real-Time Countdown Logic

- On tiket screen mount: call `socket.emit('queue:join', { slotId, antrianId })`
- Listen for `queue:update` event: `{ nomorAktif, durasiRataAktual, antrianList }`
- Compute remaining estimate client-side:
  - `const minutesLeft = (nomorUrut - nomorAktif) * (durasiRataAktual ?? estimasiDurasiMenit)`
  - Display: `±${Math.round(minutesLeft)} menit`
  - When `minutesLeft === 0`: show dipanggil state regardless of statusAntrian
- On `queue:almost` event `{ menit, pesan }`: switch countdown text to amber color
- On disconnect: show reconnecting banner, continue showing last known estimate

### Race Condition UX (QUEUE-02 requirement)

- Citizen can tap "Pilih Sesi" on a card that shows 1 slot remaining
- Backend uses `SELECT FOR UPDATE` — returns 409 if slot taken
- Frontend must handle 409 gracefully:
  1. Session card immediately flips to "penuh" visual state
  2. `<Alert variant="destructive">` appears: "Sesi ini baru saja penuh. Pilih sesi lain."
  3. No navigation occurs
  4. Alert auto-dismisses after 4 seconds
  5. Citizen can select another available session

### Form Validation Timing

- Buat Jadwal dialog: validate on blur for each field
- Date picker: validate on selection (inline — cannot type invalid date)
- Estimasi durasi: validate `min=5 max=30` on blur; show preview immediately on change
- Konfirmasi page: validate balita selection before enabling CTA (no blur needed — radio group)

### Accessibility

- Back button (ChevronLeft, Screens 1–3): `<Button aria-label="Kembali ke halaman sebelumnya" variant="ghost">`
- Calendar grid: `role="grid"`, day cells `role="gridcell"`, selected `aria-selected="true"`, disabled `aria-disabled="true"`
- Session cards: `aria-disabled="true"` on full sessions; `<Button aria-label="Pilih Sesi 1 08:00 – 09:00">`
- Queue number: `aria-label="Nomor antrian Anda: 07"`
- Countdown region: `aria-live="polite"` — updates announced to screen reader on change
- Dialog: `role="dialog"`, `aria-labelledby` pointing to dialog title
- Status badges: `role="status"`, text conveys state (not just color)
- All interactive elements: `min-h-[44px]` touch targets
- Colors: never sole conveyor of meaning — always paired with text or icon

## Executor Notes

1. **Figma verification is mandatory** before implementing each screen — pull frames `5:2314`, `5:2630`, `5:2902`, `5:3116`, `5:15526` to confirm colors, spacing, and component layouts. This spec is directionally correct; defer to Figma where it contradicts.

2. **Install shadcn components first** — run all 8 `npx shadcn add` commands listed in the Design System section before writing component code.

3. **Countdown "estimasi" rule is hard** — never replace "±XX menit" with a clock time like "08:14". The UI must always say estimate, never absolute time. This is a product requirement (CLAUDE.md + QUEUE-03).

4. **Race condition (QUEUE-02)** — the frontend does NOT prevent double-tap. The backend SELECT FOR UPDATE is the guard. Frontend must handle 409 gracefully on the confirmation screen, not just the session list.

5. **Socket.IO room** — join room `sesi:{slotId}` on TiketAntrianPage mount. Unsubscribe on unmount. Do NOT leave the room on browser tab hide (user may switch tabs and come back).

6. **useAntrianStore (Zustand)** — holds transient navigation state only: selected date, selected slotId, selected balitaId. Clear on successful antrian creation. Do NOT store antrian data here — use TanStack Query.

7. **WA notification** — frontend never calls Fonnte directly. Show "notifikasi akan dikirim" only. The backend BullMQ handles actual delivery.

8. **Puskesmas jadwal creation** — slot generation preview is client-side (floor(60 / estimasi) × 3 sesi). The actual slot creation is server-side. Preview is only informational.

9. **Mobile-first** — all screens designed mobile-first (max-width 400px centered). The ManajemenJadwalPage switches to table layout at `sm:` breakpoint (640px+).

10. **Route registration** — remember to add all 4 citizen routes and 1 puskesmas route to `frontend/src/router/index.tsx` with ProtectedRoute wrappers and role guards.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| citizen browser → GET /api/jadwal/tersedia | Citizen-initiated; posyanduUtamaId comes from auth context |
| citizen browser → GET /api/sesi | JadwalId from navigation state |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-18 | Tampering | PilihSesiPage: jadwalId injected via navigation state manipulation | accept | The server verifies slot availability and ownership on POST /api/antrian/ambil. A manipulated jadwalId at most causes the citizen to see a different sesi list; they still cannot create antrian for a posyandu they don't belong to. |
| T-02-19 | Information Disclosure | PilihTanggalPage leaks posyandu info for other posyandu | mitigate | useJadwalTersedia sends the citizen's posyanduUtamaId from their profile; server filters by that posyanduId (D-01 enforced in backend). |
| T-02-20 | Tampering | D-02 bypass: citizen navigates directly to /citizen/antrian/pilih-sesi without posyanduUtamaId | mitigate | PilihSesiPage checks for jadwalId from state; if absent, redirects back to pilih-tanggal. Backend enforces posyanduId filter on slot creation. |
| T-02-SC | Tampering | npm/pip/cargo installs | mitigate | No new npm packages; shadcn components already installed in 02-01 |
</threat_model>

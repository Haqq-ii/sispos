Wave 2 — Backend Antrian module with SELECT FOR UPDATE race condition guard + BullMQ WA notification extension.

This plan covers QUEUE-02 (race condition), QUEUE-03 (nomorUrut formula), QUEUE-04 (Socket.IO broadcast), and QUEUE-06 (WA via BullMQ). It is the most security-critical plan in Phase 02.

Key implementation rules (from CLAUDE.md §Antrian KRITIS):
- SELECT FOR UPDATE MUST use prisma.$transaction + $queryRaw — never Prisma fluent API alone
- $queryRaw returns snake_case column names (terisi, kuota, jadwal_id) — NOT camelCase
- Socket.IO broadcast MUST happen OUTSIDE the prisma.$transaction block (after commit)
- ALWAYS guard `if (!io) return` before any `io.to(...).emit(...)`
- BullMQ enqueue ALWAYS — never call Fonnte directly (per D-05 and CLAUDE.md §WhatsApp)
- Ownership check on cancel and GET detail: verify antrian.wargaId === req.user.userId

Purpose: QUEUE-02 requires the race condition to be deterministic. This plan delivers the only place in Phase 2 where the DB lock is acquired.

Output: antrian module (3 files), antrian.schema.ts, antrian.job.ts, notification.queue.ts extended, notification.worker.ts extended, app.ts updated.

    FUNCTION ambilAntrian(slotId: string, balitaId: string, wargaId: string):
    - Uses prisma.$transaction(async (tx) => { ... }) — this is the ONLY place SELECT FOR UPDATE appears
    - Inside transaction:
      a. Execute $queryRaw: `SELECT id, kuota, terisi, jadwal_id, durasi_rata_aktual FROM slot_sesi WHERE id = ${slotId} FOR UPDATE`
      b. Result is typed as Array<{ id: string; kuota: number; terisi: number; jadwal_id: string; durasi_rata_aktual: number | null }>
      c. CRITICAL: column names are snake_case (jadwal_id, durasi_rata_aktual) NOT camelCase — this is because $queryRaw bypasses Prisma's name mapping
      d. Destructure: `const [slot] = slots` — if !slot, throw with code SLOT_TIDAK_DITEMUKAN
      e. If slot.terisi >= slot.kuota, throw with code SLOT_PENUH
      f. Lookup warga: `const warga = await tx.warga.findUnique({ where: { id: wargaId }, select: { posyanduUtamaId: true, nomorPonsel: true } })`
      g. NOTE: use `posyanduUtamaId` (NOT `posyanduId`) — this is the verified field name in prisma/schema.prisma
      h. Check for duplicate: `const existing = await tx.antrian.findUnique({ where: { slotId_balitaId: { slotId, balitaId } } })` — if exists, throw SUDAH_DAFTAR
      i. Increment terisi: `await tx.slotSesi.update({ where: { id: slotId }, data: { terisi: { increment: 1 } } })`
      j. nomorUrut = slot.terisi + 1 (the new value after increment)
      k. Create antrian: `const antrian = await tx.antrian.create({ data: { slotId, balitaId, wargaId, nomorUrut, statusAntrian: 'menunggu' }, include: { slotSesi: { include: { jadwal: { include: { posyandu: true } } } } } })`
      l. Return from transaction: { antrianId: antrian.id, nomorUrut, slotId, jadwal: antrian.slotSesi.jadwal, wargaNomorPonsel: warga?.nomorPonsel ?? '' }
    - OUTSIDE the transaction block: call broadcastQueueUpdate(slotId) and enqueueAntrianWaJob (imported from notification module)
    - Per D-03: estimasiMenit = nomorUrut × jadwal.estimasiDurasiMenit (nomorAktif = 0 in Phase 2)
    - Return the full result including estimasiMenit for the controller response

    FUNCTION batalkanAntrian(antrianId: string, wargaId: string):
    - Fetch: `const antrian = await prisma.antrian.findFirst({ where: { id: antrianId, wargaId } })`
    - If !antrian, throw with code ANTRIAN_TIDAK_DITEMUKAN (404 — per D-06; using findFirst with wargaId prevents ID enumeration)
    - If antrian.statusAntrian !== 'menunggu', throw with code TIDAK_BISA_BATALKAN
    - Update in transaction: set statusAntrian = 'dibatalkan', decrement slotSesi.terisi by 1
    - OUTSIDE transaction: call broadcastQueueUpdate(antrian.slotId)
    - Return { antrianId, statusAntrian: 'dibatalkan' }

    FUNCTION getAntrianSaya(wargaId: string):
    - Return today's antrian for this warga: prisma.antrian.findFirst({ where: { wargaId, statusAntrian: { in: ['menunggu', 'dipanggil'] }, createdAt: { gte: startOfToday } }, include: { slotSesi: { include: { jadwal: { include: { posyandu: true } } } }, balita: { select: { namaBalita: true } } } })
    - startOfToday = new Date(); startOfToday.setUTCHours(0,0,0,0)

    FUNCTION getAntrianById(antrianId: string, wargaId: string):
    - prisma.antrian.findFirst({ where: { id: antrianId, wargaId } })
    - If null, throw ANTRIAN_TIDAK_DITEMUKAN (ownership + existence check in one query)
    - Include full relations for tiket display: slotSesi (include jadwal include posyandu), balita
    - Return with estimasiMenit = nomorUrut × slotSesi.jadwal.estimasiDurasiMenit

    FUNCTION broadcastQueueUpdate(slotId: string) — async:
    - Import io from '../../config/socket'
    - FIRST LINE: `if (!io) { logger.warn('Socket.IO belum siap — skip broadcast'); return }`
    - Fetch active antrian for slot: prisma.antrian.findMany({ where: { slotId, statusAntrian: { in: ['menunggu', 'dipanggil'] } }, orderBy: { nomorUrut: 'asc' }, select: { id: true, nomorUrut: true, statusAntrian: true } })
    - Fetch slot durasiRataAktual: prisma.slotSesi.findUnique({ where: { id: slotId }, select: { durasiRataAktual: true } })
    - Emit: `io.to('sesi:' + slotId).emit('queue:update', { nomorAktif: 0, durasiRataAktual: slot?.durasiRataAktual ?? null, antrianList })`
    - Per D-03: nomorAktif is always 0 in Phase 2 (Meja 1 hadir not implemented until Phase 3)

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-09 | Tampering | Race condition / slot overbooking | mitigate | prisma.$transaction + $queryRaw SELECT ... FOR UPDATE acquires row-level lock; terisi >= kuota check inside lock; 409 SLOT_PENUH returned to loser |
| T-02-10 | Elevation of Privilege | Cancel another citizen's antrian | mitigate | batalkanAntrian: findFirst({ where: { id, wargaId: req.user.userId } }) — wrong warga gets 404, not 403, preventing ID enumeration |
| T-02-11 | Information Disclosure | GET /api/antrian/:id by different citizen | mitigate | getAntrianById: findFirst({ where: { id, wargaId } }) — unauthorized access returns 404, not 403 |
| T-02-12 | Tampering | WA phone number injection via BullMQ job | mitigate | enqueueAntrianWaJob uses warga.nomorPonsel from DB query (not from request body); attacker cannot control target number |
| T-02-13 | Elevation of Privilege | Non-citizen calls antrian endpoints | mitigate | requireRole('citizen') on all antrian routes; puskesmas/kader get 403 FORBIDDEN |
| T-02-14 | Tampering | Socket.IO broadcast before transaction commits | mitigate | broadcastQueueUpdate called AFTER prisma.$transaction resolves (outside the async callback) |
| T-02-15 | Denial of Service | POST /api/antrian/ambil flood creates jobs | accept | Behind authMiddleware + requireRole; each citizen can have only 1 active antrian (SUDAH_DAFTAR check); BullMQ jobs are idempotent. Low risk for academic project. |
| T-02-SC | Tampering | npm/pip/cargo installs | mitigate | No new npm packages; slopcheck not required |
</threat_model>

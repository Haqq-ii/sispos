<objective>
Wave 1 — Backend Jadwal + Posyandu modules.

Two new backend modules ship in this plan:
1. posyandu module: minimal GET /api/posyandu filtered by the authenticated puskesmas account (per D-08). Returns the puskesmas's own posyandu list for the BuatJadwal dropdown.

Purpose: QUEUE-01 requires Puskesmas can create jadwal with SlotSesi auto-generate. This is the complete backend for that requirement.

Output: posyandu module (3 files), jadwal module (3 files), shared jadwal Zod schema, app.ts updated to mount both routers.

    Step A — Create backend/src/modules/jadwal/jadwal.service.ts with 4 exported functions:

    Function createJadwal(data: CreateJadwalInput, puskesmasId: string):
    - SECURITY CHECK: verify the posyandu belongs to this puskesmas before creating. Query `prisma.posyandu.findFirst({ where: { id: data.posyanduId, puskesmasId } })`. If null, throw an error with code POSYANDU_TIDAK_DITEMUKAN.
    - Compute kuota = Math.floor(60 / data.estimasiDurasiMenit)
    - Define SESI_CONFIG array with 3 entries: nomorSesi 1/2/3, labelSesi "Sesi 1 (08:00 - 09:00)"/"Sesi 2 (09:00 - 10:00)"/"Sesi 3 (10:00 - 11:00)", jamMulai hours 8/9/10, jamSelesai hours 9/10/11
    - Use `prisma.$transaction(async (tx) => { ... })` to atomically:
      a. Call tx.jadwal.create with posyanduId, puskesmasId, tanggalPelaksanaan: new Date(data.tanggalPelaksanaan), estimasiDurasiMenit: data.estimasiDurasiMenit, statusJadwal: 'aktif'
      b. Build slot data: for each SESI_CONFIG, create a Date via `new Date(0)` and call `.setUTCHours(hours, 0, 0, 0)` to set time (avoids timezone pitfall — Prisma @db.Time is stored as UTC hours)
      c. Call tx.slotSesi.createMany({ data: slots }) where each slot has jadwalId, nomorSesi, labelSesi, jamMulai, jamSelesai, kuota, terisi: 0
    - Return the created jadwal including slotSesi
    - Catch Prisma error P2002 (unique constraint on [posyanduId, tanggalPelaksanaan]) and rethrow with code JADWAL_SUDAH_ADA

    Function getJadwalList(puskesmasId: string, page: number, limit: number):
    - Return paginated list: prisma.jadwal.findMany({ where: { puskesmasId }, include: { posyandu: { select: { namaPosyandu: true } }, slotSesi: { select: { id: true, nomorSesi: true, labelSesi: true, kuota: true, terisi: true } } }, orderBy: { tanggalPelaksanaan: 'desc' }, skip: (page-1)*limit, take: limit })
    - Also return total count for pagination meta

    Function getJadwalTersedia(posyanduId: string, bulan: string):
    - bulan is format 'YYYY-MM'. Build startDate = new Date(bulan + '-01') and endDate = last day of that month
    - Return jadwal dates with available slots: prisma.jadwal.findMany({ where: { posyanduId, tanggalPelaksanaan: { gte: startDate, lte: endDate }, statusJadwal: 'aktif', slotSesi: { some: { terisi: { lt: prisma.slotSesi.fields.kuota } } } }, select: { tanggalPelaksanaan: true, id: true } })
    - Note: to check terisi < kuota in a where clause use: slotSesi: { some: { terisi: { lt: Prisma.sql`slot_sesi.kuota` } } } — but this is complex. Simpler: return all aktif jadwal for the month and include slotSesi; let the client filter. OR use a raw approach. Recommended: return all jadwal with their slotSesi and let the API response include hasAvailableSlot: slotSesi.some(s => s.terisi < s.kuota).

    Function getSesiList(jadwalId: string):
    - Return prisma.slotSesi.findMany({ where: { jadwalId }, include: { jadwal: { select: { estimasiDurasiMenit: true, tanggalPelaksanaan: true, posyandu: { select: { namaPosyandu: true } } } } }, orderBy: { nomorSesi: 'asc' } })
    - Format jamMulai and jamSelesai using `.toISOString().substring(11, 16)` (UTC hours to avoid timezone pitfall) in the controller response mapping, not here in service

    Step B — Create backend/src/modules/jadwal/jadwal.controller.ts:
    - createJadwalHandler: validate with CreateJadwalSchema.safeParse, call createJadwal, return 201 or mapped error (400 for validation, 403 for POSYANDU_TIDAK_DITEMUKAN, 409 for JADWAL_SUDAH_ADA, 500 for others)

    - getJadwalTersediaHandler: read query param bulan (required), call getJadwalTersedia, return 200
    - getSesiListHandler: read query param jadwalId (required), call getSesiList, format jamMulai and jamSelesai via `.toISOString().substring(11, 16)`, return 200

    - Import `type { AuthRequest }` from './auth.middleware'
    - Export function `requireRole(...roles: RolePengguna[])` returning an Express RequestHandler
    - The handler checks: if `!req.user` or `!roles.includes(req.user.role)`, respond with 403 `{ success: false, error: 'FORBIDDEN', message: 'Anda tidak memiliki akses ke endpoint ini.' }` and return

    - It calls `void socket.join('sesi:' + slotId)` to add this socket to the room

    - The `io` module-level variable and the `initSocket()` function must NOT be changed
    - All existing disconnect handler logic must be preserved

| T-02-03 | Tampering | queue:join slotId parameter | accept | The slotId is used only for socket.join() — no DB write. A crafted slotId joins a room for a non-existent slot; no data is returned to them since the room has no emitters. Low risk. |

| Token | Value | Tailwind Class | Usage |

## Screen Contracts

### Screen 1 — Login (Single Gateway)

**Route:** `/login`
**Figma frames:** `5:5654` (Citizen), `5:5731` (Citizen alt), `5:5809` (Kader), `5:5886` (Kader alt), `5:13077` (Puskesmas)

**Layout:**
- Full-screen centered column, max-width 400px, centered horizontally
- Top: SISPOS logo (favicon.svg) + wordmark "SISPOS" at display size
- Below logo: tagline body text "Digitalisasi Posyandu Indonesia"
- Card surface (bg-green-50): rounded-lg (8px), shadow-sm, p-6, contains all form fields
- Mobile: no max-width constraint, p-4 page padding

**Role auto-detection (implemented in state, reflected in UI):**
- As user types identifier → detect format → show role badge
  - 16-digit numeric → badge "Warga (Citizen)" in green
  - Starts with 08xx / +62 (phone) → badge "Kader / Staff" in blue-600
  - Contains @ (email) → badge "Puskesmas" in purple-600
- Role badge uses `<Badge>` shadcn component, positioned inline below identifier field
- Label field: `placeholder="NIK, No HP, atau Email"` (adapts as user types)

**Fields:**
1. Identifier input: `type="text"`, label "Nomor Identitas", placeholder "NIK, No HP, atau Email"
2. Password/PIN input: `type="password"`, label "Kata Sandi / PIN", placeholder "••••••"
3. Show/hide password toggle (EyeIcon / EyeOffIcon from Lucide) inside input trailing slot
   - `aria-label="Tampilkan kata sandi"` when hidden; `aria-label="Sembunyikan kata sandi"` when visible
4. Remember me: not included (JWT httpOnly cookie handles session)
5. CTA: `<Button>` full-width, `bg-primary` (green-600), label "**Masuk ke SISPOS**"

**Primary focal point:** the green "Masuk ke SISPOS" CTA button at the bottom of the form card.

**States:**
- Loading: button shows `<Loader2 className="animate-spin" />` prefix, disabled
- Error (wrong credentials): `<Alert variant="destructive">` below form — "NIK, No HP, atau kata sandi salah. Silakan coba lagi."
- Error (account locked): destructive alert — "Akun terkunci. Silakan coba lagi dalam [X] menit."
- Error (unverified): destructive alert — "Akun belum diverifikasi. Cek WhatsApp Anda untuk kode OTP."

**Links below card:**
- "Belum punya akun? **Daftar sekarang**" — text-sm, link styled with `text-primary underline`
- Visible only when identifier appears to be NIK (citizen context)

**Interactions:**
- Auto-focus identifier field on mount
- Enter key submits form
- Role badge animates in with fade (opacity 0 → 1, 150ms) after 100ms debounce from typing

### Screen 2 — Register Citizen

**Route:** `/register`
**Figma frames:** `4:87`, `4:142`

**Layout:**
- Full-screen centered column, max-width 400px
- Back button (ChevronLeft icon) top-left, routes to `/login`
- Title "Buat Akun Baru" at display size
- Subtitle "Daftarkan diri sebagai Warga Posyandu" at body size, text-gray-500
- Form card (bg-green-50, p-6, rounded-lg)

**Fields (in order):**
1. NIK Ibu: `type="text"`, label "NIK Ibu (16 Digit)", placeholder "3471000000000000"
   - maxLength=16, inputMode="numeric"
   - Helper text (text-xs, gray-500): "NIK tertera di KTP"
   - Zod: `z.string().length(16).regex(/^\d{16}$/)`
2. Nama Lengkap: `type="text"`, label "Nama Lengkap", placeholder "Siti Rahayu"
   - Zod: `z.string().min(2).max(200)`
3. Nomor Ponsel: `type="tel"`, label "Nomor HP (WhatsApp aktif)", placeholder "081234567890"
   - Helper text: "OTP akan dikirim ke nomor ini via WhatsApp"
   - Zod: Indonesian phone regex `z.string().regex(/^(08|\+628)[0-9]{8,11}$/)`
4. Kata Sandi: `type="password"`, label "Kata Sandi", placeholder "Min. 8 karakter"
   - Password strength indicator: 3-level bar (lemah/sedang/kuat), renders below input
   - Show/hide toggle (`aria-label="Tampilkan kata sandi"` / `aria-label="Sembunyikan kata sandi"`)
   - Zod: `z.string().min(8)`
5. Konfirmasi Kata Sandi: `type="password"`, label "Ulangi Kata Sandi"
   - Real-time match validation (onBlur)
   - Zod: `.refine(data => data.katasandi === data.konfirmasi, ...)`

**CTA:** "**Daftar & Kirim OTP**" — full-width primary button

**States:**
- Loading: button disabled + spinner, label "Mengirim OTP..."
- Error (NIK exists): inline field error "NIK sudah terdaftar. Coba login."
- Error (HP exists): inline field error "Nomor HP sudah terdaftar."
- Error (server): alert destructive "Gagal mendaftar. Coba beberapa saat lagi."
- All errors appear as `<p className="text-sm font-bold text-destructive">` below the relevant field

**Primary focal point:** the NIK field at top of the form card — it anchors role detection and is the first interaction.

**Link:** "Sudah punya akun? **Masuk**" below card

## Interaction Contracts

### Form Validation Timing
- Show field errors: on `onBlur` (not real-time while typing — reduces anxiety)
- Exception: NIK length shows live counter "X/16" as user types
- Exception: Password strength shows live bar as user types
- OTP: no blur delay — validate immediately on digit entry

### Navigation Flow
```
/login
  ↓ (no account)
/register
  ↓ (submit)
/register/verifikasi
  ↓ (OTP confirmed)
/register/lokasi
  ↓ (saved or skipped)
/register/lokasi-selesai
  ↓ (5s or CTA click)
/citizen/dashboard

/login → /citizen/dashboard  (NIK login)
/login → /kader/dashboard    (HP login)
/login → /puskesmas/dashboard (email login)
```

### Protected Route Guards
- Routes `/*` (except `/login`, `/register`, `/register/*`) redirect to `/login` if `useAuthStore.isAuthenticated === false`
- Implementation: `<ProtectedRoute>` wrapper component in router

### OTP Auto-submit
- When 6th digit entered, auto-trigger submit after 300ms delay (no button click required)

### PIN Input (Kader Login)
- PIN field: `type="password"`, `inputMode="numeric"`, `maxLength=6`
- Show as 6 dots (visual masking via `font-[text-security-disc]` or CSS approach)
- On submit failure: clear all 6 dots, re-focus field, show attempt count

### Cascade Dropdown Reset
- When Provinsi changes: reset Kabupaten, Kecamatan, Kelurahan to empty
- When Kabupaten changes: reset Kecamatan, Kelurahan to empty
- When Kecamatan changes: reset Kelurahan to empty

## Accessibility

- All form inputs have associated `<Label htmlFor>` (via shadcn `<FormLabel>`)
- Error messages linked via `aria-describedby` (shadcn `<FormMessage>` handles this)
- Loading states: `aria-busy="true"` on submit buttons, `aria-disabled` on downstream dropdowns
- Color is never the sole conveyor of meaning: errors have icon (AlertCircle) + text
- Minimum contrast: all text meets WCAG AA 4.5:1 against their background
- OTP boxes: `aria-label="Digit ke-{N}"` on each input, `aria-live="polite"` region for error messages
- Lock screen: `role="alert"` on lock notice container
- Password eye icon toggle (Login + Register screens):
  - When password hidden: `<button aria-label="Tampilkan kata sandi">`

## Executor Notes

1. **Figma verification is mandatory** before implementing each screen — pull frames listed at top of this doc to confirm colors, spacing, and component layouts. The contract above is directionally correct; defer to Figma where it contradicts.
2. **shadcn install first** — run all 9 `npx shadcn add` commands listed in Design System section before writing any component code.
3. **react-hook-form + zod required** — add to package.json before form implementation.
4. **OTP boxes are custom** — do not use a third-party OTP library without security vetting.
5. **Countdown timer** — implement as a custom hook with `setInterval`, clean up on unmount.
6. **Role detection logic** in `LoginPage.tsx` must mirror backend detection: 16 numeric chars = citizen, phone pattern = kader, email pattern = puskesmas.
7. **Never call Fonnte directly from frontend** — OTP send is backend-only via BullMQ.
8. **JWT cookie** — do not store token in localStorage; Axios interceptor reads httpOnly cookie automatically. `useAuthStore` holds only the user metadata, not the token.

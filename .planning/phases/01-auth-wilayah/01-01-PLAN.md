  <action>
    Step 1 — Create backend/src/modules/auth/auth.service.ts with the following exported async functions. Do NOT use string concatenation for queries — use Prisma ORM exclusively.

    issueTokens(userId: string, role: string): { accessToken: string; refreshToken: string }
      Signs accessToken with jwt.sign({ userId, role }, env.JWT_SECRET, { expiresIn: env.JWT_ACCESS_EXPIRY }).
      Signs refreshToken with jwt.sign({ userId }, env.JWT_REFRESH_SECRET, { expiresIn: env.JWT_REFRESH_EXPIRY }).
      Returns both tokens as plain strings.

    registerWarga(data: z.infer<typeof RegisterSchema>): Promise<{ nomorPonselMasked: string }>
      Check unique: if (await prisma.warga.findUnique({ where: { nikIbu: data.nikIbu } })) throw error with code 'NIK_SUDAH_TERDAFTAR'.
      Check unique: if (await prisma.warga.findUnique({ where: { nomorPonsel: data.nomorPonsel } })) throw error with code 'HP_SUDAH_TERDAFTAR'.
      Hash password: bcrypt.hash(data.password, env.BCRYPT_ROUNDS).
      Create Warga: prisma.warga.create with nikIbu, namaLengkap, nomorPonsel, passwordHash, statusVerifikasi: 'belum_verifikasi'.
      Generate OTP: crypto.randomInt(100000, 999999).toString() — use Node's built-in crypto module.
      Expire time: new Date(Date.now() + 5 * 60 * 1000).
      Invalidate old OTPs: prisma.otp.updateMany({ where: { nomorPonsel, tujuan: 'registrasi', sudahDipakai: false }, data: { sudahDipakai: true } }).
      Save new Otp record: prisma.otp.create with nomorPonsel, kodeOtp, tujuan: 'registrasi', sudahDipakai: false, kedaluwarsaPada.
      Enqueue: await enqueueOtpJob(data.nomorPonsel, kodeOtp).
      Return masked phone: replace digits 4-8 with '****' (e.g., "081234567890" → "0812****7890").

    sendOtp(nomorPonsel: string): Promise<void>
      Find Warga by nomorPonsel — throw 'WARGA_TIDAK_DITEMUKAN' if not found.
      Generate OTP, invalidate old, save new, enqueue (same logic as registerWarga OTP section).

    verifyOtpAndLogin(nomorPonsel: string, kodeOtp: string): Promise<{ tokens: { accessToken: string; refreshToken: string }; user: { id: string; namaLengkap: string; role: 'citizen' } }>
      Find valid OTP: prisma.otp.findFirst({ where: { nomorPonsel, kodeOtp, sudahDipakai: false, kedaluwarsaPada: { gt: new Date() }, tujuan: 'registrasi' } }).
      If not found: throw error with code 'OTP_TIDAK_VALID'.
      Mark sudahDipakai: prisma.otp.update({ where: { id: otp.id }, data: { sudahDipakai: true } }).
      Update Warga status: prisma.warga.update({ where: { nomorPonsel }, data: { statusVerifikasi: 'terverifikasi' } }).
      Return issueTokens(warga.id, 'citizen') + user object { id, namaLengkap, role: 'citizen' }.

    updateLokasi(userId: string, data: z.infer<typeof UpdateLokasiSchema>): Promise<void>
      prisma.warga.update({ where: { id: userId }, data: { provinsi, kabupaten, kecamatan, kelurahan, rw: data.rw ?? null, rt: data.rt ?? null } }).

    Step 2 — Create backend/src/modules/auth/auth.controller.ts:
    Each handler follows the response contract: { success: true, data: {...}, message: "..." } or { success: false, error: "KODE_ERROR", message: "Pesan Bahasa Indonesia" }.

    registerHandler: POST body validated with RegisterSchema.safeParse. On WARGA NIK/HP conflict return 409. On success return 201.
    otpSendHandler: POST body validated with OtpSendSchema. Return 200.
    otpVerifyHandler: POST body validated with OtpVerifySchema. On OTP_TIDAK_VALID return 400. On success: set cookies then return 200.
      Set access_token cookie: res.cookie('access_token', tokens.accessToken, { httpOnly: true, sameSite: 'lax', secure: env.NODE_ENV === 'production', maxAge: 15 * 60 * 1000 }).
      Set refresh_token cookie: res.cookie('refresh_token', tokens.refreshToken, { httpOnly: true, sameSite: 'lax', secure: env.NODE_ENV === 'production', maxAge: 7 * 24 * 60 * 60 * 1000 }).
    updateLokasiHandler: manually verify JWT from req.cookies.access_token using jwt.verify(token, env.JWT_SECRET) as { userId: string }. If no cookie or invalid, return 401. Then call updateLokasi service.

    Use try/catch in each handler. Unknown errors return 500 with error: 'INTERNAL_ERROR'.

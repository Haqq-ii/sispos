    STEP 2 — dashboard.service.ts:

    Create backend/src/modules/dashboard/dashboard.service.ts. Define and export:

    Interface StuntingMapPoint: { posyanduId: string; namaPosyandu: string; kelurahan: string; lat: number; lng: number; total: number; breakdown: Record<string, number> }

    Interface DashboardStats: { totalPemeriksaan: number; totalBalita: number; breakdown: Record<string, number> }

    Function getStuntingMapData(puskesmasId: string, bulan: string): Promise<StuntingMapPoint[]>:
    - Parse bulan ('YYYY-MM') into startOfMonth (new Date(year, month-1, 1)) and endOfMonth (new Date(year, month, 0, 23, 59, 59))
    - Query prisma.posyandu.findMany where puskesmasId equals the passed puskesmasId (from JWT, not client), select id/namaPosyandu/kelurahan/latitude/longitude, nested include: jadwal (where tanggalPelaksanaan between start and end) > slotSesi > antrian (where statusAntrian 'selesai') > pemeriksaan (select statusGizi only)
    - Filter result to posyandu with latitude !== null and longitude !== null
    - Flatten the nested chain to get all pemeriksaan, build breakdown Record by statusGizi using reduce
    - Return mapped array of StuntingMapPoint

    Function getDashboardStats(puskesmasId: string, bulan: string): Promise<DashboardStats>:
    - Same date range from bulan
    - Reuse the same findMany query structure
    - Flatten all pemeriksaan, count total and breakdown
    - Count distinct wargaId from antrian in the period as totalBalita (use a separate prisma.antrian.findMany and reduce to unique wargaIds, or groupBy if available)
    - Return DashboardStats

    STEP 3 — dashboard.controller.ts:

    Create backend/src/modules/dashboard/dashboard.controller.ts. Define two async handler functions (not exported directly — used by routes):

    getStuntingMapHandler(req: AuthRequest, res: Response): void
    - Extract bulan from req.query.bulan as string; default to current month in YYYY-MM format if absent: new Date(Date.now() + 7*60*60*1000).toISOString().slice(0,7)
    - puskesmasId = req.user!.userId (NEVER from query params — IDOR guard per T-04-01-01)
    - Call getStuntingMapData(puskesmasId, bulan), respond 200 { success: true, data: result }
    - Catch: respond 500 { success: false, error: 'INTERNAL_ERROR', message: 'Gagal memuat data peta stunting.' }

    getDashboardStatsHandler(req: AuthRequest, res: Response): void
    - Same puskesmasId extraction from JWT
    - Same bulan extraction
    - Call getDashboardStats(puskesmasId, bulan), respond 200 { success: true, data: result }

    STEP 4 — dashboard.routes.ts:

    Create backend/src/modules/dashboard/dashboard.routes.ts. Export dashboardRouter (Router from express). Register:
      GET /stunting → authMiddleware, requireRole('puskesmas'), getStuntingMapHandler
      GET /stats → authMiddleware, requireRole('puskesmas'), getDashboardStatsHandler

    STEP 5 — app.ts registration:

    In backend/src/app.ts, add the import for dashboardRouter and register:
      import { dashboardRouter } from './modules/dashboard/dashboard.routes'
      app.use('/api/dashboard', dashboardRouter)

    Place the registration after the existing rekapHarianRouter line (before the 404 handler) to maintain consistency with the existing pattern.

    Export async function checkAndIncrementRateLimit(wargaId: string): Promise<void>:
    - today = new Date(Date.now() + 7*60*60*1000).toISOString().slice(0, 10)  // WIB date per Pitfall 5
    - key = 'chatbot:gizi:citizen:${wargaId}:${today}'
    - count = await redis.incr(key)
    - If count === 1:
        endOfDayWIB = new Date(Date.now() + 7*60*60*1000)
        endOfDayWIB.setUTCHours(17, 0, 0, 0)  // 17:00 UTC = 00:00 WIB next day
        if endOfDayWIB.getTime() <= Date.now(): endOfDayWIB.setUTCDate(endOfDayWIB.getUTCDate() + 1)
        await redis.expireat(key, Math.floor(endOfDayWIB.getTime() / 1000))
    - If count > DAILY_LIMIT: throw Object.assign(new Error('Batas 20 pesan per hari telah tercapai. Coba lagi besok.'), { code: 'RATE_LIMIT_EXCEEDED' })

    Export async function chatGizi(wargaId: string, message: string): Promise<string>:
    - Call checkAndIncrementRateLimit(wargaId) — throws before any OpenAI call if over limit
    - Load history: prisma.riwayatChat.findMany({ where: { wargaId }, orderBy: { createdAt: 'desc' }, take: 10, select: { role: true, pesan: true } }), then .reverse()
    - Graceful degradation: if !process.env.OPENAI_API_KEY → return 'Asisten AI tidak tersedia saat ini. Silakan hubungi posyandu langsung.'
    - Lazy import OpenAI: const OpenAI = (await import('openai')).default; const openai = new OpenAI({ apiKey: process.env.OPENAI_API_KEY })
    - Build messages: [{ role: 'system', content: GIZI_SYSTEM_PROMPT }, ...history.map(h => ({ role: h.role as 'user' | 'assistant', content: h.pesan })), { role: 'user', content: message }]
    - Call openai.chat.completions.create({ model: 'gpt-4o', temperature: 0.6, max_tokens: 300, messages })
    - reply = response.choices[0]?.message?.content ?? 'Maaf, tidak bisa memproses permintaan saat ini.'
    - Persist both turns: await prisma.riwayatChat.createMany({ data: [{ wargaId, role: 'user', pesan: message }, { wargaId, role: 'assistant', pesan: reply }] })
    - Return reply

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → Express API | wargaId must come from JWT (req.user!.userId); never accept wargaId from request body |
| OpenAI API | System prompt hardcoded server-side; user message treated as data not instructions |
| Redis rate limit | Key includes wargaId from JWT; client cannot manipulate the counter key |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-03-01 | Spoofing | chatGizi — wargaId for rate limit and RiwayatChat | mitigate | wargaId extracted from req.user!.userId (JWT) in chatGiziHandler; never from req.body |
| T-04-03-02 | Denial of Service | OpenAI billing via chatbot spam | mitigate | checkAndIncrementRateLimit called BEFORE openai.chat.completions.create; 20/day hard cap enforced atomically via Redis INCR |
| T-04-03-03 | Tampering | Prompt injection in user message | mitigate | GIZI_SYSTEM_PROMPT hardcoded server-side; user input appended as data (role: 'user') — AI treats it as question, not instruction override |
| T-04-03-04 | Information Disclosure | Chat history leakage | mitigate | RiwayatChat query uses WHERE wargaId = req.user!.userId — citizen cannot access another citizen's history |
| T-04-03-05 | Tampering | Role field in RiwayatChat | mitigate | Only 'user' and 'assistant' roles stored (VarChar(10) constraint); tool_call messages are transient (per Pitfall 6) |
</threat_model>

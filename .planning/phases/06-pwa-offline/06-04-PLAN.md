    === STEP 2: usePwaStore.ts (new Zustand store for PWA install state — D-16) ===

    Create `frontend/src/stores/usePwaStore.ts`:

    Define the BeforeInstallPromptEvent interface:
    ```
    interface BeforeInstallPromptEvent extends Event {
      prompt(): Promise<void>
      readonly userChoice: Promise<{ outcome: 'accepted' | 'dismissed' }>
    }
    ```

    Export this interface for use in App.tsx.

    State interface: `{ deferredPrompt: BeforeInstallPromptEvent | null; setDeferredPrompt(e: BeforeInstallPromptEvent | null): void; triggerInstall(): Promise<void> }`

    `triggerInstall`: gets deferredPrompt from state; if null returns; calls `await deferredPrompt.prompt()`; awaits `deferredPrompt.userChoice`; if outcome === 'accepted', calls setDeferredPrompt(null).

    Export: `export const usePwaStore = create<...>((set, get) => ({ ... }))`

    === STEP 3: App.tsx — OfflineBanner + beforeinstallprompt (D-07, D-16) ===

    Rewrite App.tsx (currently 12 lines, minimal — full rewrite is safe):

    New imports:
    - `import { useEffect } from 'react'`
    - `import { BrowserRouter } from 'react-router-dom'`
    - `import { AppRouter } from '@/router'`
    - `import { Toaster } from '@/components/ui/toaster'`
    - `import { OfflineBanner } from '@/components/offline/OfflineBanner'`
    - `import { useOfflineStatus } from '@/hooks/useOfflineStatus'`
    - `import { usePwaStore, type BeforeInstallPromptEvent } from '@/stores/usePwaStore'` (if BeforeInstallPromptEvent is exported from store) OR define it inline in App.tsx

    App component:
    ```
    export default function App() {
      const { setDeferredPrompt } = usePwaStore()
      const isOnline = useOfflineStatus()

      useEffect(() => {
        const handler = (e: Event) => {
          e.preventDefault()
          setDeferredPrompt(e as BeforeInstallPromptEvent)
        }
        window.addEventListener('beforeinstallprompt', handler)
        return () => window.removeEventListener('beforeinstallprompt', handler)
      }, [setDeferredPrompt])

      return (
        <BrowserRouter>
          <OfflineBanner />
          {!isOnline && <div className="h-10" aria-hidden="true" />}
          <AppRouter />
          <Toaster />
        </BrowserRouter>
      )
    }
    ```

    The `<div className="h-10" aria-hidden="true" />` creates a 40px spacer in normal flow when the fixed OfflineBanner is visible, preventing Meja page content from being hidden under the banner (per UI-SPEC §1).

    If `BeforeInstallPromptEvent` is not exported from usePwaStore, define it locally in App.tsx with the same interface shape.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser event → usePwaStore | beforeinstallprompt captured via e.preventDefault(); stored as UI state |
| vite.config.ts BackgroundSync → SW | Workbox generates SW at build time; BackgroundSync shorthand in runtimeCaching options (not direct BackgroundSyncPlugin import) |
| SW BackgroundSync → API | SW-layer retry sends JWT cookie (httpOnly, sent automatically by browser); backend validates per request |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-04-01 | Elevation of Privilege | beforeinstallprompt capture | accept | e.preventDefault() prevents browser default install prompt hijack; our handler only stores the event; no elevated permissions granted by PWA install |
| T-06-04-02 | Tampering | BackgroundSync order in vite.config.ts | mitigate | Specific patterns inserted BEFORE catch-all (Pitfall 1); wrong order causes all API calls to match catch-all which has no BackgroundSync — no security issue but sync won't work |
| T-06-04-03 | Information Disclosure | OfflineBanner reveals connectivity state to page-level JS | accept | navigator.onLine is already publicly accessible to any script; OfflineBanner adds no new information exposure |
| T-06-04-04 | Repudiation | Workbox BackgroundSync retry after page close | mitigate | SW-layer retry still sends JWT cookie; if cookie expired, 401 response stops retry and logs to sync_errors; maxRetentionTime=24h limits retry window |
| T-06-SC | Tampering | npm install idb@8 (from Plan 06-01) | mitigate | Handled in 06-01 Plan via checkpoint:human-verify |
</threat_model>

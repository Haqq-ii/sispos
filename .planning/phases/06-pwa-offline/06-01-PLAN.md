    Step A — Create `frontend/src/hooks/useOfflineSync.ts` (D-02, D-04, D-06, D-10):

    Imports: useState, useEffect, useCallback, useRef from 'react'; useToast from '@/hooks/use-toast'; useQueryClient from '@tanstack/react-query'; useKaderMejaStore from '@/stores/useKaderMejaStore'; apiClient from '@/lib/axios'; getOfflineDB, logSyncError, countPending, generateTempId from '@/lib/offline-db'.

    Export `useOfflineSync()` hook returning `{ pendingCount: number, enqueueOperation, syncAll }`.

    pendingCount: useState(0). On mount, call refreshCount().

    refreshCount (useCallback, no deps): calls countPending() → setPendingCount.

    enqueueOperation (useCallback): signature `(queueType: 'kehadiran' | 'pemeriksaan' | 'meja5', payload: Record<string,unknown>) => Promise<void>`.
    - Awaits getOfflineDB().
    - Writes payload to correct store: `${queueType}_queue` (kehadiran_queue | pemeriksaan_queue | meja5_queue) using `db.add(storeName, payload)`.
    - Calls refreshCount().

    syncAll (useCallback): async function. If `!navigator.onLine` return early.
    Full sync logic per the architecture diagram in RESEARCH.md:
    1. `const db = await getOfflineDB()`
    2. `let skipCount = 0`
    3. Process kehadiran_queue FIFO (getAllFromIndex 'by_timestamp'):
       - For each item: call `apiClient.patch('/antrian/' + item.antrianId + '/' + item.action)`.
       - On success: `await db.delete('kehadiran_queue', item.id)`.
       - On 422 or 409 response error: `await logSyncError(item, err.response?.status ?? 0, err.message)` + skipCount++. Do NOT rethrow — skip and continue (D-04).
    4. Process pemeriksaan_queue FIFO:
       - Maintain `const tempIdMap: Record<string,string> = {}` for temp→real ID resolution.
       - For type='create': `apiClient.post('/growth/pemeriksaan', item.data)` → `realId = response.data.data.id` → `tempIdMap[item.tempPemeriksaanId] = realId` → call `setActivePemeriksaanId(realId)` from useKaderMejaStore → `db.delete('pemeriksaan_queue', item.id)`.
       - For type='patch-tanda-klinis' or 'patch-catatan': `resolvedId = tempIdMap[item.tempPemeriksaanId] ?? item.tempPemeriksaanId` → `apiClient.patch('/growth/pemeriksaan/' + resolvedId, item.data)` → `db.delete('pemeriksaan_queue', item.id)`.
       - On 422/409 error: logSyncError + skipCount++.
    5. Process meja5_queue FIFO:
       - type='immunization': `apiClient.post('/immunization', item.data)`.
       - type='selesai': `apiClient.patch('/antrian/' + item.data.antrianId + '/selesai')`.
       - On success: `db.delete('meja5_queue', item.id)`.
       - On error: logSyncError + skipCount++.
    6. Toast (per D-10 + UI-SPEC Copywriting):
       - skipCount === 0: `toast({ description: 'Data berhasil disinkronkan' })`.
       - skipCount > 0: `toast({ description: 'Gagal sinkronkan ' + skipCount + ' data — lihat rekap harian', variant: 'destructive' })`.
    7. `await queryClient.invalidateQueries({ queryKey: ['antrian', 'kader'] })`.
    8. `await refreshCount()`.

    Wire 'online' event listener using the ref pattern (avoids stale closure):
    - `const syncAllRef = useRef(syncAll)`.
    - `useEffect(() => { syncAllRef.current = syncAll }, [syncAll])`.
    - `useEffect(() => { const handler = () => { void syncAllRef.current() }; window.addEventListener('online', handler); return () => window.removeEventListener('online', handler) }, [])`.

    Note: All API calls in syncAll use try/catch per-item (not a single outer try). Individual errors must not abort the full batch — only the failed item is logged to sync_errors and skipped (D-04).

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → IndexedDB | Queue writes happen in-browser; no server trust boundary at write time |
| IndexedDB → API | On sync, queued items sent via JWT-authenticated axios requests to backend |
| Browser event → App | beforeinstallprompt + online/offline events from browser runtime |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Tampering | IndexedDB queue stores | mitigate | Backend Zod validation + IDOR guards run on every synced request; malformed IDB payloads rejected at API layer (ASVS V5) |
| T-06-02 | Information Disclosure | pemeriksaan_queue / kehadiran_queue stores | accept | catatanKlinis stored plaintext by design (D-13); device lock is physical security layer; JSDoc note added in offline-db.ts per UU PDP awareness |
| T-06-03 | Repudiation | syncAll() → API on session expiry | mitigate | axios.ts JWT refresh interceptor handles 401 → refresh → retry; if refresh fails, queue items remain for next login rather than causing mid-sync logout |
| T-06-04 | DoS | pemeriksaan_queue retry flooding | mitigate | D-04: skip-on-422 pattern + logSyncError prevents infinite retry per item; Workbox maxRetentionTime=24h bounds SW-layer retry window |
| T-06-05 | Elevation of Privilege | crypto.randomUUID() in HTTP context | mitigate | generateTempId() tries crypto.randomUUID() first; falls back to timestamp+Math.random() composite string if unavailable (A3 — localhost is secure context in Docker dev) |
| T-06-SC | Tampering | npm install idb@8 | mitigate | Package tagged [ASSUMED] for slopcheck; blocking checkpoint:human-verify (Task 1) required before install; idb cross-verified via npm view + official GitHub jakearchibald/idb |
</threat_model>

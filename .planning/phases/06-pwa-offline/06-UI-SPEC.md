## Component Specifications

### 1. OfflineBanner (new component)

**File:** `frontend/src/components/offline/OfflineBanner.tsx`

```
Behavior: Renders only when navigator.onLine === false
          Listens to window 'online' and 'offline' events
          Sticky position: top of viewport, below no other sticky element
          Does NOT push page content down — position: fixed or sticky above page body
```

**Visual spec:**
- Container: `fixed top-0 left-0 right-0 z-50 bg-orange-600 text-white`
- Height: `py-2.5 px-4` — auto height to fit single line
- Content: `flex items-center justify-center gap-2`
- Icon: `WifiOff` from Lucide, `h-4 w-4 text-orange-200`
- Text: `"Mode Offline — data tersimpan lokal"` — 12px, weight 600, white
- When visible: page body receives `pt-[40px]` offset to prevent content being hidden under banner

**States:**
- Online: component returns `null` (unmounted)
- Offline: renders full-width orange bar

### 4. PWA Install Button (new element in App.tsx / KaderDashboardPage)

**Placement:** KaderDashboardPage header, right side (where "Tukar Meja" normally does not appear). If screen is too narrow, position as a small floating bottom-right FAB on dashboard only.

Preferred placement: inside the green header bar, far-right, same row as the user name/role display.

**Visual spec:**
- Button: `<button className="bg-[rgba(0,166,62,0.6)] border border-[rgba(0,201,80,0.5)] rounded-xl px-3 py-1.5 text-white text-xs font-medium flex items-center gap-1.5">`
- Icon: `Download` from Lucide, `h-3.5 w-3.5`
- Text: `"Pasang Aplikasi"`
- Min height: 44px (touch target — use `min-h-[44px]` or `py-3` if header allows)
- Visibility: only when `deferredPrompt !== null` (beforeinstallprompt captured and not yet used)
- After install: button disappears (set `deferredPrompt = null`, hide state)
- Never shown if already running in standalone mode (`window.matchMedia('(display-mode: standalone)').matches`)

**Interaction:**
1. `window.addEventListener('beforeinstallprompt', e => { e.preventDefault(); setDeferredPrompt(e) })`
2. On button click: `deferredPrompt.prompt()` → await `userChoice` → clear `deferredPrompt`
3. No custom install page (D-17 locked)

## Interaction States

### Submit while Offline (Meja 1–5)

```
User taps submit button
  ↓
Mutation interceptor detects navigator.onLine === false
  ↓
Write to IndexedDB store (kehadiran_queue / pemeriksaan_queue / meja5_queue)
  ↓
Toast: "Tersimpan lokal, akan sync saat online" (default variant, duration 3000ms)
  ↓
SyncPendingBadge counter increments
  ↓
UI navigates forward normally (same as online success path)
```

### Auto-sync on Reconnect

```
window 'online' event fires
  ↓
Read all IndexedDB queue stores (FIFO order per store)
  ↓
For each queued item: POST/PATCH to API
  ↓
On item success: remove from IndexedDB, decrement badge
  ↓
On item 422/conflict: log to sync_errors store, skip (D-04)
  ↓
When all processed:
  - If all succeeded: toast "Data berhasil disinkronkan"
  - If some failed: toast "Gagal sinkronkan {N} data — lihat rekap harian"
  ↓
SyncPendingBadge shows 0 → hides
```

### Meja 4 Offline Feature Degradation

```
navigator.onLine === false
  ↓
"Rekam Suara" button: disabled + opacity-50 + Tooltip "Tidak tersedia offline"
"Analisis AI" button: disabled + opacity-50 + Tooltip "Tidak tersedia offline"
catatanKlinis textarea: fully active (no change)
  ↓
User types catatan manually → saves → goes to IndexedDB (via offline intercept)
  ↓
On sync: rekomendasiAi = null, catatanSTT = null in PATCH payload (D-14)
```

Output: Modified Meja1Page.tsx and Meja2Page.tsx with offline branches at each critical mutation call site.
</objective>

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-02-01 | Tampering | kehadiran_queue payload | mitigate | antrianId comes from the server-fetched antrian list (TanStack Query); not user-editable; backend IDOR guard validates kader owns the antrian |
| T-06-02-02 | Tampering | tempPemeriksaanId chain | accept | tempId is a local UUID generated client-side; backend never receives a tempId directly — sync resolves to real ID via POST first; any injected tempId in pemeriksaan_queue that matches no real POST results in a failed PATCH (404) logged to sync_errors |
| T-06-02-03 | Spoofing | Meja 2 offline bypasses Z-Score validation | accept | Z-Score is informational for kader — the backend computes it independently during sync; skipping the display offline is a UI-only concession, not a security bypass |

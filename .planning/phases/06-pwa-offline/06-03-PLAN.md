<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Meja 4 STT/AI → disabled offline | STT and AI are network-dependent; disabling prevents misleading error states |
| catatan textarea → pemeriksaan_queue | plaintext catatan stored offline; encrypted at backend on sync |
| selesai offline → store reset | Store cleared client-side offline; server-side countdown update deferred to sync |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-03-01 | Information Disclosure | catatanKonsultasi in pemeriksaan_queue offline | accept | Plaintext by design (D-13); device lock is physical security; backend encrypts on sync per UU PDP |
| T-06-03-02 | Tampering | Meja 4 AI/STT disabled — attacker removes disable | accept | Attacker would need to modify JS at runtime; STT and AI endpoints still validate JWT + IDOR at server; no server-side data change from disabling |
| T-06-03-03 | Tampering | selesai offline skips durasiRataAktual update | accept | Moving average update is a business metric, not a security boundary; sync will deliver the PATCH when online; Socket.IO broadcast deferred |
| T-06-03-04 | Repudiation | Meja 3/4/5 queue entries with temp pemeriksaanId | mitigate | Sync resolution (POST creates real ID, PATCH substitutes) is handled in useOfflineSync.syncAll(); failed resolution logged to sync_errors per D-06 |
</threat_model>

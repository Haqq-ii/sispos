    queue-kader.service.ts: add selesaikanAntrian(antrianId, kaderId):
      From 03-RESEARCH.md §Pattern 7 — full implementation provided there. Key rules:
      1. prisma.$transaction with SELECT FOR UPDATE (identical to hadir pattern):
           const rows = await tx.$queryRaw`SELECT id, "statusAntrian", "slotId", "waktuMulaiLayanan" FROM antrian WHERE id = ${antrianId} FOR UPDATE`
           if (!rows[0]) throw {code:'ANTRIAN_TIDAK_DITEMUKAN'}
           if (rows[0].statusAntrian !== 'dipanggil') throw {code:'ANTRIAN_BELUM_AKTIF'}
      2. Set waktuSelesai = new Date()
      3. Compute durasiLayananBaru = waktuMulaiLayanan ? (waktuSelesai - waktuMulaiLayanan) / 60000 : null
      4. tx.antrian.update({where:{id}, data:{statusAntrian:'selesai', waktuSelesai}})
      5. Moving average (only if durasiLayananBaru > 0 && < 60):
           n = await tx.antrian.count({where:{slotId, statusAntrian:'selesai'}})
           slot = await tx.slotSesi.findUnique({where:{id:slotId}, select:{durasiRataAktual:true}})
           oldAvg = slot?.durasiRataAktual ?? durasiLayananBaru
           newAvg = n <= 1 ? durasiLayananBaru : (oldAvg * (n - 1) + durasiLayananBaru) / n
           await tx.slotSesi.update({where:{id:slotId}, data:{durasiRataAktual:newAvg}})
      6. AFTER tx commit:
           void broadcastQueueUpdate(slotId)         // CLAUDE.md §Antrian: WAJIB di luar tx
           enqueue BullMQ 'wa:selesai' job
      7. Posyandu IDOR guard: verify antrian.slotSesi.jadwal.posyanduId === kader.posyanduId

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → PATCH /api/antrian/:id/selesai | antrianId from URL; kader triggers from Meja 5 button |
| Browser → GET /api/reports/rekap-harian | slotId + format query params |
| rekap-harian → ExcelJS cell content | namaBalita string from DB |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-07-01 | Tampering | Concurrent selesai calls for same antrian (double-selesai) | mitigate | SELECT FOR UPDATE in prisma.$transaction; second call finds statusAntrian='selesai' → throws ANTRIAN_BELUM_AKTIF (not 'dipanggil') |
| T-03-07-02 | Tampering | Excel formula injection via namaBalita (e.g., namaBalita = "=CMD(...)") | mitigate | sanitize function prefixes with ' any cell value starting with =, +, -, @ before ExcelJS addRow call |
| T-03-07-03 | Denial of Service | PDF generation with very large dataset (>1000 rows) | accept | Per-slot antrian is bounded by kuota (floor(60/7) ≈ 8-12 per sesi × 3 sesi = ~36 max). Not a realistic DoS vector. |
| T-03-07-04 | Spoofing + IDOR | slotId in rekap-harian belongs to another posyandu | mitigate | rekapHarianHandler verifies slotSesi.jadwal.posyanduId matches kader's posyanduId before generating report |
| T-03-07-05 | Non-repudiation | Moving average tampered with post-hoc via direct DB update | accept | Correct approach requires application-level audit. Academic scope — no additional controls beyond AuditLog already in place for Pemeriksaan. |
| T-03-07-SC | Tampering | npm installs | mitigate | No new installs; ExcelJS and pdfkit already in backend/package.json |
</threat_model>

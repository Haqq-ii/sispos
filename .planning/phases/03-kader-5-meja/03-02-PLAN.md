    growth.routes.ts — follow antrian.routes.ts pattern exactly:

      POST /api/growth/pemeriksaan → authMiddleware → requireRole('kader','ketua_kader') → createPemeriksaanHandler

      ERROR_MAP: { BALITA_TIDAK_DITEMUKAN: 404, ANTRIAN_TIDAK_DITEMUKAN: 404, PEMERIKSAAN_SUDAH_ADA: 409 }

      antrianId: z.string().uuid().optional()
      balitaId: z.string().uuid({ required_error: 'balitaId wajib diisi' })
      beratBadan: z.number().positive().max(100)  // max 100 allows extreme values with confirmation header
      tinggiBadan: z.number().positive().max(200).optional()
      lingkarKepala: z.number().positive().optional()
      lingkarLengan: z.number().positive().optional()
      catatanKonsultasi: z.string().optional()

    growth.service.ts — createPemeriksaan:
      1. Fetch balita from DB to get jenisKelamin, tanggalLahir
      2. Validate BB range: if beratBadan > 30 AND req.headers['x-konfirmasi-biologis'] !== 'true' → throw {code:'VALIDASI_BIOLOGIS_PERLU_KONFIRMASI'}
      3. Compute ageMonths = ageInMonths(balita.tanggalLahir, now)
      4. Compute zScoreBbU = computeZScore('WAZ', sex, beratBadan, ageMonths)
      5. Compute zScoreTbU = tinggiBadan ? computeZScore('HAZ', sex, tinggiBadan, ageMonths) : null
      6. Compute zScoreBbTb = (tinggiBadan && beratBadan) ? computeZScore('WHZ', sex, beratBadan, tinggiBadan) : null
      7. statusGizi = determineStatusGizi(zScoreBbU, zScoreTbU, zScoreBbTb)
      8. catatanKonsultasiEnc = catatanKonsultasi ? encrypt(catatanKonsultasi) : null
      9. prisma.$transaction: tx.pemeriksaan.create({...fields, catatanKonsultasi: catatanKonsultasiEnc}) THEN tx.auditLog.create({ userId: kaderId, userRole: 'kader', aksi: 'CREATE_PEMERIKSAAN', tabelTerkait: 'pemeriksaan', recordId: pemeriksaan.id, dataSebelum: null, dataSesudah: {beratBadan, tinggiBadan, zScoreBbU, zScoreTbU, zScoreBbTb, statusGizi}, ipAddress: req.ip, userAgent: req.headers['user-agent'] })
      CRITICAL: Do NOT include catatanKonsultasi or rekomendasiAi in dataSesudah AuditLog
      10. Return pemeriksaan (with decrypted catatanKonsultasi in response but encrypted in DB)

    getPemeriksaanHistory: prisma.pemeriksaan.findMany({where:{balitaId}, orderBy:{tanggalPemeriksaan:'asc'}, select:{id,tanggalPemeriksaan,beratBadan,tinggiBadan,zScoreBbU,zScoreTbU,zScoreBbTb,statusGizi,statusGiziOverride}})
    Note: catatanKonsultasi and rekomendasiAi are NOT included in history response (encrypted fields, not needed for chart)

      GET  /api/kader/active-meja         → getActiveMejaHandler
      PATCH /api/kader/active-meja        → setActiveMejaHandler  (body: {mejaNumber: 1-5, slotId: uuid})
      DELETE /api/kader/active-meja       → clearActiveMejaHandler
      GET  /api/kader/slot/:slotId/antrian → getSlotAntrianHandler
      PATCH /api/antrian/:id/hadir        → hadirAntrianHandler
      PATCH /api/antrian/:id/tangguhkan   → tangguhkanAntrianHandler

    queue-kader.service.ts:

    getActiveMeja(kaderId): redis.get(`kader:${kaderId}:activeMeja`) → parse "mejaNumber:slotId" → return {activeMeja, slotId} | null
    setActiveMeja(kaderId, mejaNumber, slotId): redis.set(key, `${mejaNumber}:${slotId}`, 'EX', 86400)
    clearActiveMeja(kaderId): redis.del(`kader:${kaderId}:activeMeja`)

    getSlotAntrian(slotId): prisma.antrian.findMany({where:{slotId, statusAntrian:{in:['menunggu','dipanggil','ditangguhkan']}}, include:{balita:{select:{namaBalita,jenisKelamin,tanggalLahir}}, warga:{select:{rt}}}, orderBy:{nomorUrut:'asc'}})

    hadirAntrian(antrianId, kaderId):
      prisma.$transaction: SELECT FOR UPDATE on antrian row → validate statusAntrian IN ('menunggu','ditangguhkan') → update {statusAntrian:'dipanggil', waktuCheckin:now, waktuMulaiLayanan:now}
      AFTER tx (CRITICAL per CLAUDE.md §Antrian): void broadcastQueueUpdate(antrian.slotId)
      AFTER tx: enqueue BullMQ WA job 'wa:dipanggil'
      broadcastQueueUpdate is the function from antrian.service.ts — import it; do NOT duplicate

    tangguhkanAntrian(antrianId, kaderId):
      prisma.$transaction: SELECT FOR UPDATE → validate statusAntrian === 'dipanggil' → update {statusAntrian:'ditangguhkan'}
      AFTER tx: void broadcastQueueUpdate(slotId)

    goShowAntrian(slotId, balitaId, wargaId, kaderId):
      prisma.$transaction: SELECT FOR UPDATE on slotSesi → validate terisi < kuota → create Antrian({slotId, balitaId, wargaId, nomorUrut: next available, isDaftarManual: true, statusAntrian:'menunggu'}) → slotSesi.update({terisi: {increment:1}})
      AFTER tx: void broadcastQueueUpdate(slotId)

    Redis key ownership: `kader:${kaderId}:activeMeja` — kaderId comes from req.user.userId (JWT, not request body). Never trust client-provided kaderId.

    Kader IDOR guard: for hadirAntrian/tangguhkanAntrian, verify that antrian.slotSesi.jadwal.posyanduId === kader.posyanduId before operating.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → /api/growth/pemeriksaan | Untrusted BB/TB values, balitaId from request body |
| Client → /api/kader/active-meja | mejaNumber + slotId from request body |
| Client → /api/antrian/:id/hadir | antrianId from URL param |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-02-01 | Spoofing + IDOR | GET/PATCH /api/growth/pemeriksaan — kader accesses pemeriksaan of another posyandu | mitigate | growth.service.ts verifies antrian.slotSesi.jadwal.posyandu.id matches req.user kader's posyanduId |
| T-03-02-02 | Tampering | Z-Score value provided in request body | mitigate | Backend computes Z-Score from who-growth-tables.json; any Z-Score in request body is silently ignored |
| T-03-02-03 | Non-repudiation | AuditLog bypass for pemeriksaan INSERT | mitigate | AuditLog.create in same prisma.$transaction as pemeriksaan.create — both commit or both rollback |
| T-03-02-04 | Spoofing | Redis key kader:{kaderId}:activeMeja — client provides fake kaderId | mitigate | kaderId taken from req.user.userId (JWT verified by authMiddleware), never from request body |
| T-03-02-05 | Tampering | Concurrent hadir calls for same antrian (race condition) | mitigate | SELECT FOR UPDATE on antrian row inside prisma.$transaction prevents double-hadir |
| T-03-02-06 | Information Disclosure | catatanKonsultasi stored as plaintext | mitigate | encrypt() called in growth.service.ts before tx.pemeriksaan.create; field is never stored as UTF-8 text |
| T-03-02-SC | Tampering | npm installs | mitigate | All packages installed in 03-01 (pre-audited [OK]); no new installs in this plan |
</threat_model>
